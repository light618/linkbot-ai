# 抖音开放平台能力申请清单

## 🎯 需要开通的能力

根据 LinkBot-AI 的功能需求，需要在抖音开放平台申请以下能力：

### ✅ 必需能力（必须申请）

#### 1. **用户信息授权 (user_info)**
- **能力描述**：获取用户昵称、头像、open_id 等基础信息
- **用途**：
  - OAuth 授权后获取用户基本信息
  - 展示授权账号信息
  - 多账号管理
- **文档链接**：https://developer.open-douyin.com/docs/resource/zh-CN/dop/develop/openapi/account-permission/userinfo
- **审核难度**：⭐⭐☆ (简单)

#### 2. **视频相关能力 (video.list, video.data)**
- **能力描述**：获取视频列表、视频详情
- **用途**：
  - 获取账号下的所有视频
  - 监听短视频评论
  - 选择要监控的视频
- **文档链接**：https://developer.open-douyin.com/docs/resource/zh-CN/dop/develop/openapi/video-management/introduction
- **审核难度**：⭐⭐☆ (简单)

#### 3. **评论管理 (video.comment)**
- **能力描述**：获取评论列表、发送评论回复
- **用途**：
  - **获取视频评论**：轮询获取新评论
  - **自动回复评论**：AI 智能回复
  - **评论筛选**：过滤垃圾评论
- **文档链接**：https://developer.open-douyin.com/docs/resource/zh-CN/dop/develop/openapi/comment-management/introduction
- **审核难度**：⭐⭐⭐☆ (中等)

### 🔥 核心能力（重点申请）

#### 4. **互动消息推送 (im_room)**
- **能力描述**：接收直播间评论、点赞、关注等互动消息
- **用途**：
  - **监听直播间评论**：实时接收用户消息
  - **监听互动行为**：点赞、关注、送礼等
  - **自动化处理**：触发自动回复或人工介入
- **文档链接**：https://developer.open-douyin.com/docs/resource/zh-CN/dop/develop/openapi/live-management/introduction
- **审核难度**：⭐⭐⭐⭐☆ (较难)
- **申请建议**：
  - 强调应用场景：智能客服系统
  - 提供应用截图和说明
  - 可考虑先上线其他功能后再申请

#### 5. ~~**私信发送 (im_send)**~~ ⚠️ 已下架
- **状态**：❌ 抖音开放平台已不支持该能力
- **说明**：抖音已关闭私信发送接口，显示"暂不支持"
- **替代方案**：
  - 通过评论回复引导用户关注账号
  - 在视频简介中提供联系方式
  - 引导用户添加微信/企业微信
  - 在评论区设置置顶回复提供联系方式

### 📋 申请优先级

## 🔧 申请步骤

### 步骤1：登录开放平台
1. 访问：https://developer.open-douyin.com/
2. 登录企业账号

### 步骤2：进入能力接入
1. 进入 **"控制台"** → **"应用管理"**
2. 找到应用 `LinkBot-AI 智能客服系统`
3. 点击 **"能力接入"** 或 **"申请能力"**

### 步骤3：申请权限
1. 在能力列表中，找到需要的权限
2. 点击 **"申请"** 按钮
3. 填写申请理由（参考下方模板）
4. 提交审核

### 步骤4：等待审核
- **审核时间**：1-3个工作日
- **审核结果**：通过 / 驳回
- **审核反馈**：会通过站内信通知

### 替代方案：评论区线索转化
```
应用场景：通过评论回复引导用户留资

1. AI自动识别意向用户
   - 识别"价格"、"联系方式"、"怎么买"等关键词
   - 自动回复引导语："感谢关注，详细资料已私信给您"

2. 引导用户关注账号
   - 在回复中引导："关注账号，获取更多优惠信息"
   - 利用抖音的自动回复功能在关注后发送信息

3. 评论区收集线索
   - 引导用户在评论区留言"1"或"资料"
   - 通过关键词筛选意向用户
```

## ⚠️ 注意事项

### 审核不通过的常见原因
1. **应用描述不够清晰** - 需要详细说明使用场景
2. **缺少应用截图** - 提供完整的应用界面截图
3. **能力申请过多** - 分批申请，逐步提升信任度
4. **未提供承诺** - 明确承诺不违规使用

### 提高通过率的建议
1. **先申请基础能力** - user_info、video.comment
2. **明确应用场景** - 详细描述业务场景
3. **提供截图证明** - 应用已开发完成
4. **做出合规承诺** - 遵守平台规则
5. **联系客服沟通** - 如有疑问及时咨询
